## PE Dormant Export Analyzer

This tool cross-references runtime execution logs with a Windows Portable Executable (PE) file's structural metadata. By parsing the export directory and Structured Exception Handling (SEH) unwind data (`.pdata`), it identifies contiguous blocks of unused or dormant functions that meet a minimum capacity threshold. 

This is useful for binary auditing, code dead-space elimination, and telemetry verification.

### 🌟 Key Features
* **Linear PE Structure Mapping:** Parses disk-backed DLLs using `pefile` to retrieve the Relative Virtual Address (RVA) and exact byte boundaries of exported symbols.
* **Unwind Data Correlation:** Leverages the `.pdata` section (Exception Directory) to calculate function sizes based on actual stack unwind configurations, falling back safely if explicit structures are missing.
* **Multi-Layer Exclusion Filtering:** Filters out active code paths dynamically via JSONL runtime telemetry logs, alongside an optional flat-file checklist of disallowed APIs.
* **Contiguous Block Identification:** Grouping algorithm walks functions sequentially by memory address to find unbroken, adjacent "clusters" of unused code sections that meet specified byte requirements.

---

### ⚙️ How It Works
1. **Size Mapping:** The tool maps virtual addresses to their memory sizes by reading the start and end offsets inside the PE file's SEH exception directory.
2. **Export Profiling:** All exports are indexed, correlated to their computed sizes, and sorted linearly by their position in memory.
3. **Unsafe Evaluation:** The application reads telemetry data and flags any function captured during runtime monitoring or listed in the unsafe definitions file.
4. **Gap Analysis:** The sorted array is processed to discover sequential blocks of uncalled exports, summing their byte sizes to verify if they can accommodate the target capacity.

### 🚀 Usage

#### Prerequisites
~~~bash
pip install pefile argparse
~~~

#### Command Line Arguments
| Argument | Description | Required | Default |
| :--- | :--- | :--- | :--- |
| `-d`, `--dll` | Absolute path to the local target DLL file under evaluation. | **Yes** | N/A |
| `-t`, `--telemetry` | Path to the runtime JSONL log file generated by monitoring tools. | **Yes** | N/A |
| `-s`, `--size` | The target capacity threshold required for the contiguous block (in bytes). | **Yes** | N/A |
| `-u`, `--unsafe` | Path to a text file containing line-separated API names to force exclude. | No | `None` |

#### Examples

**Find contiguous unused zones of at least 512 bytes within a target DLL:**
~~~bash
python export_analyzer.py -d "C:\Windows\System32\uxtheme.dll" -t active_telemetry.jsonl -s 512
~~~

**Enforce strict api exclusions while seeking a 1024-byte block:**
~~~bash
python export_analyzer.py -d "C:\Windows\System32\kernel32.dll" -t active_telemetry.jsonl -u sensitive_apis.txt -s 1024
~~~
