## Process Stability Campaign Harness

This script functions as an automated testing engine designed to execute and monitor application stability campaigns. By parsing multi-process execution profiles generated by your plan creator, it tracks application behavior, confirms that required target DLL dependencies load cleanly into memory, injects companion evaluation payloads, and processes complex kernel crash codes.

### 🌟 Key Features
* **Automated Profile Orchestration:** Sequentially runs through testing runs configured within a target JSON campaign plan.
* **Lazy-Loading Dependency Tracking:** Actively queries active memory via Windows `tasklist` bindings to confirm target modules have successfully loaded before proceeding with the evaluation loop.
* **Sidecar Payload Linkage:** Dynamically matches target PIDs to arguments, launching sidecar engineering utilities concurrently alongside the active process.
* **Aggressive Crash Mitigation:** Includes built-in environment sanitization—such as wiping the Office Registry Resiliency tree for Microsoft Word (`winword.exe`) to prevent diagnostic blocks from hanging on safe-mode prompts.
* **Kernel Fault Detection:** Intercepts and logs critical process outcomes, separating natural process terminations from critical system faults like Access Violations (`0xC0000005`) and invalid C-Runtime closures.

---

### ⚙️ How It Works
1. **Environment Purging:** Prior to starting an iteration, the script forcefully terminates lingering target processes from memory and scrubs application safety locks.
2. **Process Bootstrapping:** It spawns the primary target application process context and tracks its tracking handle (`process.pid`).
3. **Verification Delay:** It loops until the targeted hook module becomes active inside the process space, then allows a short initialization period to settle the environment.
4. **Utility Ingestion:** If a companion tool is specified, it formats the string context to dynamically insert the current host process `{pid}` and triggers the verification process.
5. **Log Streaming:** The program logs structural runtime duration records, chronological timestamps, and specific return parameters directly into JSON Lines (`.jsonl`) results manifests.

### 🚀 Usage

#### Prerequisites
~~~bash
# Built completely using Python standard libraries (subprocess, json, os, time, argparse)
python stability-harness.py --help
~~~

#### Command Line Arguments
| Argument | Description | Required | Default |
| :--- | :--- | :--- | :--- |
| `-p`, `--plan` | Absolute path to the multi-profile target configuration JSON plan. | **Yes** | N/A |
| `-l`, `--log` | Destination file for appending step-by-step test execution logs. | No | `execution_telemetry_results.jsonl` |

#### Execution Example
~~~bash
python stability-harness.py -p plans/stability_matrix.json -l logs/campaign_run_01.jsonl
~~~
